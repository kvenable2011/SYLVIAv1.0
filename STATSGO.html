Description
A characterization of the data, including its intended use and limitations.
Abstract: This dataset (STATSGO_Set1 and STATSGO_Set2) represents the soil characteristics within individual, local NHDPlusV2 catchments and upstream, contributing watersheds based on the STATSGO dataset (see Data Sources for links to NHDPlusV2 data and STATSGO data). Attributes were calculated for every local NHDPlusV2 catchment and accumulated to provide watershed-level metrics.
This data set is derived from the STATSGO landscape rasters for the conterminous USA. Individual rasters (Landscape Layers) of organic material (om), permeability (perm), water table depth (wtdep), depth to bedrock (rckdep), percent clay (clay), and percent sand (sand) were used to calculate soil characteristics for each NHDPlusV2 catchment.
The soil characteristics were summarized to produce local catchment-level and watershed-level metrics as a continuous data type (see Data Structure and Attribute Information for a description). The STATSGO data are distributed in two sets, STATSGO_Set1 and STATSGO_Set2, based on common NoData locations in each set of soil GIS layers (see ftp://newftp.epa.gov/EPADataCommons/ORD/NHDPlusLandscapeAttributes/StreamCat/Documentation/ReadMe.html).
Purpose: These data were created in support of the US EPAs National Aquatic Resource Survey (NARS) and Safe and Sustainable Waters Resources (SSWR) Programs. However, these watershed summaries of landscape data have broad applicability for hydrological, geomorphic, or ecological modelling, management, and restoration. For details on EPA NARS and SSWR Programs see http://water.epa.gov/type/watersheds/monitoring/aquaticsurvey_index.cfm and http://www.epa.gov/research/progressreport/water.htm
Supplemental information: STATSGO was split into Set_1 and Set_2 based on the no data values that exist in the rasters across each of the landscape layers. (See ReadMe file)
All code and scripts used to produce these data were written in the python or R programming languages and are freely available at: ftp://newftp.epa.gov/EPADataCommons/ORD/NHDPlusLandscapeAttributes/StreamCat/Documentation/Code/
WtDep - Seasonal water table depth of soils (cm).
Om - Organic matter content of soils (% by weight).
Perm - Permeability of soils (inches/hour).
RckDep - Depth to bedrock of soils (cm).
Clay - Clay content of soils (%).
Sand - Sand content of soils (%).
Glossary of Terms
Stream Segment: NHD line describing a section of contiguous stream or river between upstream and downstream tributaries (as depicted in USGS topographic quadrangles), except where the segment is a headwater or terminal stream (e.g., an outlet to the ocean).
Catchment (Cat): The portion of the landscape where surface flow drains directly to an NHD stream segment, excluding any upstream contributions.
Watershed (Ws): The set of hydrologically-connected catchments, consisting of all upstream catchments that contribute flow to any local catchment.
COMID: Unique identifier for each stream segment, and associated catchment, in NHDPlusV2. Watershed metrics describe entire area of all catchments that accumulate to the specific catchment based on the NHD PlusFlow table.
Riparian Buffer (Rp100): Spatial buffer of 100-m that was added to NHDPlusV2 stream network and NLCD water pixels. The buffers were used to spatially constrain calculations for some landscape metrics to within 100 meters of the riparian area.
Landscape Layer: Any geospatial layer from which we derived statistical summaries for both local catchments and watersheds. These landscape layers were comprised of several GIS formats, including vector (points, lines, or polygons) and raster.

Overview
Summary of the information content of the data, including other references to complete descriptions of entity types, attributes, and attribute values for the data.

STATSGO_Set2 contains data for Om, Perm, RckDep, and WtDep. The difference between the two exists to clarify the description of NO DATA cells throughout the data sets in table distribution. Process Steps
Information about events, parameters, tolerances and techniques applied to construct or derive the data.
Process step information
Process Step 1
Process description: Dataset downloaded from internal FTP provided by James Falcone of USGS. Data source, date, units and resolution were recorded in a project tracking spreadsheet and verified. ftp://newftp.epa.gov/EPADataCommons/ORD/NHDPlusLandscapeAttributes/StreamCat/Documentation/DataDictionary.html
Process date: 2014
Process Step 2
Process description: Projection of dataset verified as USGS Albers Equal Area Conic and projected if necessary
Process date: 2014
Process Step 3
Process description: Performed a visual inspection of dataset for gaps, edges other anomalous features. Verified how no-data values were represented and recoded values as no-data where appropriate (for instance, if no-data locations represented by 0's or -9999, these were converted to no-data values).
Process date: 2014
Process Step 4
Process description: Ran zonal statistics python script (StreamCat.py) to calculate zonal statistics for all NHDPlusV2 catchments in each NHDPlusV2 Vector Processing Unit (VPU). Zonal statistics were perofrmed using either ZonalStatisticsAsTable, TabulateArea, or spatially joining vector point data to catchments. Each method applies to the type of landscape layer being evaluated, Continuous, Categorical, or Point. We also calculated the percent of each catchment that had coverage by a given landscape layer (i.e. was not null). These results are accumulated using the NHDPlusV2 flow table to summarize upstream catchment values and complete watershed values for the metric.
Process date: 2014
Process Step 5
Process description: Clean and standardize results of zonal statistics table from accumulation script / process (makeFinalTables.py). Clean up consisted of calculating mean value of metric per zone, renaming and sorting zonal statistics fields, and writing zonal results to .csv file.
Process date: 2014

CSDGM is the US Federal Metadata standard. The Federal Geographic Data Committee originally adopted the CSDGM in 1994 and revised it in 1998. According to Executive Order 12096 all Federal agencies are ordered to use this standard to document geospatial data created as of January, 1995. The standard is often referred to as the FGDC Metadata Standard and has been implemented beyond the federal level with State and local governments adopting the metadata standard as well. The Biological Data Profile broadens the application of the CSDGM so that it is more easily applied to biological data that are not explicitly geographic (laboratory results, field notes, specimen collections, research reports) but can be associated with a geographic location. Includes taxonomical vocabulary. The Shoreline Data Profile addresses variability in the definition and mapping of shorelines by providing a standardized set of terms and data elements required to support metadata for shoreline and coastal data sets.
